## 1. Context: The "Actual" Icarus
The **actual** Icarus project is a sophisticated voice-enabled assistant leveraging:
*   **SenseVoice:** High-fidelity transcription with integrated emotion detection (Happy, Sad, Angry, etc.).
*   **Gemma 3 12B:** Multimodal reasoning engine.
*   **Kokoro TTS:** Low-latency localized speech synthesis.

Currently, Icarus uses a static **System Prompt** to define its persona (witty, sardonic, professional). While functional, it lacks **Identity Persistence**—it cannot truly "evolve" or adjust its internal state based on long-term interaction dynamics.

## 2. The DDA Scaffold Advantage
The `dda_scaffold` provides a **Meta-Cognitive Layer** that can transform Icarus from a prompt-driven bot into a state-driven agent.

### 2.1 PID Control for Persona
Instead of a static persona, we can use a **PID (Proportional-Integral-Derivative) Controller** to manage a `rigidity` or `wit_level` variable.
*   **Setpoint:** The target "wittiness" level.
*   **Process Variable:** Current wittiness detected in generated tokens.
*   **Feedback Signal:** User's detected emotion (via SenseVoice).
*   **Result:** If the user sounds "Angry" or "Sad," the PID loop automatically reduces the `wit_level` (sardonic humor) and increases `helpfulness` (professionalism) in real-time.

### 2.2 Vector Space Identity (The Attractor)
`dda_scaffold` treats identity as a "persistent attractor" in vector space. 
*   **Implementation:** We can maintain a `latent_state` vector for Icarus that gets updated with every exchange. 
*   **Cognitive Load:** Instead of just sending a "Transcript" to Gemma, we send a **State-Aware Prompt** generated by the DDA Scaffold, which includes the current "emotional coordinates" of the assistant.

### Proposed Changes to `Backend/main.py`:
*   **Replace `ICARUS_SYSTEM_PROMPT`:** With a dynamic `get_dda_prompt(session_state)` function.
*   **Emotional Feedback Loop:** Hook the `EMOTION_MAP` output from SenseVoice directly into the DDA PID controller. 
    *   *Example:* `ANGRY` user -> `DDA_Rigidity += 0.5` -> Prompt becomes "Strictly Professional."
*   **Memory Salience:** Use the DDA memory ledger to retrieve specific past interactions based on "Surprise" levels rather than just the last 10 turns.
